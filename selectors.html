Working With Selectors

Selectors

Selector Structure Description & Use Case Notes
Type <type>
type { }		 Mathces the type of the element in the HTML document. Unless specified otherwise in the CSS file, browser, or 'user agent', will apply default formatting to these tags.
Universal * { } Applies to everything on the page Usually placed at the top of the CSS file
Class <h1 class="class1">
.class1 { }		 Applies to a specific class as defined in the HTML. Classes can be used multiple times. One of the most common ways to select an element.
Multiple classes <h1 class="class1 class2">
.class1 { }
.class2 { }		 Applying multiple classes to an HTML element. Can be used to apply further enhancements to specific elements in a particular class.
ID <span id="id1">
#id1 { }		 Applies to a specific ID as defined in the HTML. Unlike classes, IDs can be only be used once. Very hard to override. Best practice recommends avoiding using ID unless absolutely necessary.

Specificity

Specificity is the order by which the browser decides which CSS styles will be displayed. A best practice in CSS is to style elements while using the lowest degree of specificity so that if an element needs a new style, it is easy to override.

In order of least specific to most specific:

type → class → chain → ID

This is done by combining multiple selectors, which we will refer to as chaining. For instance, if there was a special class for <h1> elements, the CSS would look like below:

h1.special {
  color: blue;
}

Descendent Combinators

Selecting elements that are nested within other HTML elements. For example,

.class1 li {
  font-weight: bold;
}

Multiple Selectors

Allows you to be more efficient in your code by appling the same style to multiple selectors at one time.

h1,
.class1,
#id1 {
  font-family: Tahoma;
}
